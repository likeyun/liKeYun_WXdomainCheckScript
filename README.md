# 微信域名检测自动化 HTTP 服务

## 📝 项目简介

这是一个基于 **Python + Windows 微信客户端** 的微信域名检测自动化服务。

通过调用微信内部网页访问流程，并分析微信返回页面内容，自动判断目标域名在微信环境中的访问状态。

支持检测：

-   正常访问
-   已停止访问该网页
-   将要访问
-   如需浏览，请长按网址复制后使用浏览器访问
-   非官方网页
-   未备案等限制页面

# ✨ 功能特点


## 支持

-   HTTP API 调用
-   单域名检测
-   URL 链接检测
-   自动识别微信拦截页面
-   JSON 格式返回
-   IPv4 / IPv6 支持
-   可部署为远程检测服务

## 优点

-   检测结果来自微信真实访问环境
-   检测准确
-   无需第三方检测接口

## 缺点

-   需要保持 Windows 微信客户端运行
-   单次检测约 3\~5 秒
-   需要 Windows 环境挂机运行

# 🔧 技术原理

检测流程：

    用户请求
      |
      |
    HTTP API
      |
      |
    Python检测服务
      |
      |
    Windows微信客户端
      |
      |
    微信内部浏览器访问目标URL
      |
      |
    解析微信返回页面
      |
      |
    输出检测结果

程序通过分析微信返回页面：

-   msg
-   Desc
-   Url

判断域名状态。

# 📝 检测结果 

| 颜色 | 页面 Title | 页面 Desc | 检测结果 | 
|------|------------|-----------|----------| 
| 🔴 红色拦截 | 已停止访问该网页 | 网页包含 XXX 内容，为维护绿色上网环境，已停止访问。 | 拦截 | 
| 🔵 蓝色封禁 | 将要访问 | 非微信官方网页，请确认是否继续访问。 | 拦截 | 
| ⚪ 白色封禁 | 如需浏览，请长按网址复制后使用浏览器访问 | 页面 URL | 拦截 |

# 📦 API 返回示例

## 被拦截

``` json
{
  "code": -1,
  "msg": "拦截",
  "ret": {
    "desc": [
      "将要访问"
    ]
  },
  "url": "ttjpg.top"
}
```

## 正常

``` json
{
  "code": 0,
  "msg": "正常",
  "url": "baidu.com"
}
```

# 🚀 使用方法

## 1. 登录微信电脑版

测试版本：

    微信 4.1.11.24

要求：

-   微信保持登录状态
-   微信客户端正常运行
-   Windows 不退出程序

## 2. 启动 HTTP 服务

执行：

``` bash
python check.py
```

启动后：

    http://127.0.0.1:8000

## 3. 浏览器调用

访问：

    http://127.0.0.1:8000/?url=baidu.com

或者：

    http://127.0.0.1:8000/?url=https://example.com/path

即可返回检测结果。
